# Manual Regression Test Plan — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended for Tasks 1–12 and 14–17) or superpowers:executing-plans to implement this plan task-by-task. **Task 13 (first-run execution) MUST be executed inline by the controller — do not delegate to a subagent. See note in Task 13.** Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the reusable manual regression suite (`docs/manual-regression.md`) and its vitest assertions (`tests/regression-suite.test.ts`), then execute the Required tier once against current main to find bugs prior rounds missed.

**Architecture:** Three deliverables produced in sequence: (1) a three-tier checklist file (Required ~50 / Extended ~80 / Exhaustive ~70) with `Action / Expected (declared upfront) / Actual / Verdict` per item, (2) a vitest test file that codifies mechanically-checkable items as assertions so they fail in CI on regression, (3) a date-stamped findings doc with each finding triaged. Tasks 1–12 build content (subagent-friendly, mechanical). Task 13 is the first execution (inline only — author judgment is the load-bearing input). Tasks 14–17 outline lower-priority tiers, update the README, and merge.

**Tech Stack:**
- Markdown for checklist + findings docs.
- TypeScript + vitest for assertions (already the project standard; see `tests/auto-processor.test.ts` for the established pattern).
- `glob` (Node built-in via `fs.readdirSync` recursion) + `gray-matter` (in `package.json` deps) for file-system iteration in tests.
- The shipped `harness` CLI (`dist/cli/index.js`) for execution.
- Local Ollama with `qwen3:1.7b` already pulled, per CLAUDE.md §9.

**Reference spec:** `docs/specs/2026-04-30-manual-regression-design.md`. Item numbering, tier structure, persona definitions, and acceptance criteria all come from there. When this plan and the spec disagree, the spec wins; flag the discrepancy as a finding.

```markdown
# Manual Regression Suite

> **Purpose:** Pre-committed checklist of behaviors the harness MUST satisfy at every release. Each item has an `Expected` block declared up front; the moment `Actual` differs, that's a finding — never rationalized as PASS. See `docs/specs/2026-04-30-manual-regression-design.md` for the design rationale.

## How to use this file

1. Pick a tier (Required / Extended / Exhaustive).
2. Open a fresh terminal in a clean working directory (`/tmp/regression-<date>/`).
3. For each item: read **Expected** first, run **Action**, fill in **Actual**, mark **Verdict** (PASS / FAIL / SURPRISE).
4. Findings (FAIL or SURPRISE) get logged in `docs/manual-regression-findings-YYYY-MM-DD.md`. Do not edit this file — it stays clean for re-use.
5. Mechanical assertions (no `<!-- L0:` markers, schema validity, etc.) ALSO run in `tests/regression-suite.test.ts`. Run `npm test -- regression-suite` before any release.

Every item follows this template:

**Concern:** boot/scaffold | identity-loading | skills-loading | session-learn | export | mcp | long-running | doctor | anti-pattern | persona

**Action:**
\\\`\\\`\\\`bash
{exact commands, copy-pasteable}
\\\`\\\`\\\`

**Expected:**
- {bullet list of expected behaviors}
- {include exact file paths, frontmatter fields, content fragments}
- {include exit code expectation}

**Actual (this run):** {filled at runtime — leave blank in committed version}

**Verdict (this run):** PASS | FAIL | SURPRISE

**Notes:** {anything unexpected — feeds new items}
```

```typescript
import { describe, it, expect } from 'vitest';
import { readdirSync, readFileSync, statSync, existsSync } from 'fs';
import { join } from 'path';
import matter from 'gray-matter';

const REPO_ROOT = join(__dirname, '..');
const DEFAULTS_RULES = join(REPO_ROOT, 'defaults/rules');
const DEFAULTS_SKILLS = join(REPO_ROOT, 'defaults/skills');
const TEMPLATES_DEV = join(REPO_ROOT, 'templates/dev/defaults');

const LEGACY_L0 = /<!--\s*L0:/;
const LEGACY_L1 = /<!--\s*L1:/;

function listRuleFiles(dir: string): string[] {
  if (!existsSync(dir)) return [];
  return readdirSync(dir)
    .filter((f) => f.endsWith('.md'))
    .map((f) => join(dir, f));
}

function listSkillEntries(dir: string): string[] {
  if (!existsSync(dir)) return [];
  return readdirSync(dir)
    .filter((d) => statSync(join(dir, d)).isDirectory())
    .map((d) => join(dir, d, 'SKILL.md'))
    .filter((p) => existsSync(p));
}

function listAllMarkdownRecursive(dir: string): string[] {
  if (!existsSync(dir)) return [];
  const out: string[] = [];
  for (const entry of readdirSync(dir)) {
    const full = join(dir, entry);
    if (statSync(full).isDirectory()) {
      out.push(...listAllMarkdownRecursive(full));
    } else if (entry.endsWith('.md')) {
      out.push(full);
    }
  }
  return out;
}

describe('regression-suite: shipped defaults have no legacy L0/L1 markers', () => {
  it('every file in defaults/rules/*.md has no <!-- L0: --> in body', () => {
    for (const path of listRuleFiles(DEFAULTS_RULES)) {
      const content = readFileSync(path, 'utf-8');
      expect(content, path).not.toMatch(LEGACY_L0);
      expect(content, path).not.toMatch(LEGACY_L1);
    }
  });

  it('every SKILL.md in defaults/skills/*/ has no <!-- L0: --> in body', () => {
    for (const path of listSkillEntries(DEFAULTS_SKILLS)) {
      const content = readFileSync(path, 'utf-8');
      expect(content, path).not.toMatch(LEGACY_L0);
      expect(content, path).not.toMatch(LEGACY_L1);
    }
  });

  it('every markdown file in templates/dev/defaults/ has no <!-- L0: --> in body', () => {
    for (const path of listAllMarkdownRecursive(TEMPLATES_DEV)) {
      const content = readFileSync(path, 'utf-8');
      expect(content, path).not.toMatch(LEGACY_L0);
      expect(content, path).not.toMatch(LEGACY_L1);
    }
  });
});

describe('regression-suite: shipped defaults have description: in frontmatter', () => {
  it('every rule in defaults/rules/*.md has description ≥ 20 chars', () => {
    for (const path of listRuleFiles(DEFAULTS_RULES)) {
      const { data } = matter(readFileSync(path, 'utf-8'));
      expect(data.description, `${path} missing description:`).toBeTruthy();
      expect(String(data.description).length, `${path} description too short`).toBeGreaterThanOrEqual(20);
    }
  });

  it('every skill in defaults/skills/*/SKILL.md has description ≥ 20 chars', () => {
    for (const path of listSkillEntries(DEFAULTS_SKILLS)) {
      const { data } = matter(readFileSync(path, 'utf-8'));
      expect(data.description, `${path} missing description:`).toBeTruthy();
      expect(String(data.description).length, `${path} description too short`).toBeGreaterThanOrEqual(20);
    }
  });
});

describe('regression-suite: scaffold structure', () => {
  it('templates/dev/defaults/ uses only post-collapse primitive directories', () => {
    if (!existsSync(TEMPLATES_DEV)) return;
    const allowed = new Set(['rules', 'skills', 'memory', 'intake']);
    const subdirs = readdirSync(TEMPLATES_DEV)
      .filter((entry) => statSync(join(TEMPLATES_DEV, entry)).isDirectory());
    const offenders = subdirs.filter((d) => !allowed.has(d));
    expect(offenders, `templates/dev/defaults/ contains legacy primitive dir(s): ${offenders.join(', ')}`).toEqual([]);
  });

  it('package.json exposes only bin, no library surface', () => {
    const pkg = JSON.parse(readFileSync(join(REPO_ROOT, 'package.json'), 'utf-8'));
    expect(pkg.bin).toBeDefined();
    expect(pkg.bin.harness).toBe('./dist/cli/index.js');
    expect(pkg.main, 'package.json has "main" — anti-pattern violation').toBeUndefined();
    expect(pkg.types, 'package.json has "types" — anti-pattern violation').toBeUndefined();
    expect(pkg.exports, 'package.json has "exports" — anti-pattern violation').toBeUndefined();
  });
});

describe('regression-suite: skill frontmatter schema validity', () => {
  it('every default skill has Agent Skills name + metadata fields', () => {
    for (const path of listSkillEntries(DEFAULTS_SKILLS)) {
      const { data } = matter(readFileSync(path, 'utf-8'));
```
