# Manual Regression Suite

> **Purpose:** Pre-committed checklist of behaviors the harness MUST satisfy at every release. Each item has an `Expected` block declared up front; the moment `Actual` differs, that's a finding — never rationalized as PASS. See `docs/specs/2026-04-30-manual-regression-design.md` for the design rationale.

## How to use this file

1. Pick a tier (Required / Extended / Exhaustive).
2. Open a fresh terminal in a clean working directory (`/tmp/regression-<date>/`).
3. For each item: read **Expected** first, run **Action**, fill in **Actual**, mark **Verdict** (PASS / FAIL / SURPRISE).
4. Findings (FAIL or SURPRISE) get logged in `docs/manual-regression-findings-YYYY-MM-DD.md`. Do not edit this file — it stays clean for re-use.
5. Mechanical assertions (no `<!-- L0:` markers, schema validity, etc.) ALSO run in `tests/regression-suite.test.ts`. Run `npm test -- regression-suite` before any release.

## Item format

Every item follows this template:

```
### {tier-prefix}-{NN} — {one-line title}

**Lens:** A (CLI sweep) | B (file review) | C (persona walkthrough) | combinations
**Concern:** boot/scaffold | identity-loading | skills-loading | session-learn | export | mcp | long-running | doctor | anti-pattern | persona

**Action:**
```bash
{exact commands, copy-pasteable}
```

**Expected:**
- {bullet list of expected behaviors}
- {include exact file paths, frontmatter fields, content fragments}
- {include exit code expectation}

**Actual (this run):** {filled at runtime — leave blank in committed version}

**Verdict (this run):** PASS | FAIL | SURPRISE

**Notes:** {anything unexpected — feeds new items}
```

The committed version of this file leaves Actual/Verdict/Notes blank. Each execution copies the relevant items into the current findings doc and fills them there.

**Lens:** A + B
**Concern:** boot/scaffold

```bash
mkdir -p /tmp/r-01 && cd /tmp/r-01 && rm -rf test-agent
harness init test-agent
ls -la test-agent/
find test-agent -type d | sort
find test-agent -maxdepth 1 -type f -name "*.md" | sort
